# Support

This document explains where to ask for help.

## Questions about articles

Use GitHub Discussion linked to article.

Examples:

- "I did not understand this step"
- "Do you have an alternative example?"
- "Does this also work in this case?"
- "Is there a source to explore further?"

## Site bugs

Open issue using bug report template.

Examples:

- broken page
- broken link
- accessibility problem
- build error
- rendering issue

## Article proposals

You can:

- open pull request directly with article
- open proposal issue if you want feedback before writing
- use Discussion if you want to validate relevance of topic

## Security issues

Do not open public issue.

Read `SECURITY.md`.

## Behavior or moderation issues

Read:

- `CODE_OF_CONDUCT.md`
- `MODERATION_POLICY.md`

Use private channel when report includes personal data, harassment, or sensitive content.

## Unsupported requests

Project does not guarantee:

- ongoing personal technical support
- private review of code unrelated to repository
- free consulting
- automatic promotion of products/services/events
- automatic content publication
