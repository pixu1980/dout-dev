# Open Source Checklist

Practical checklist to make dout.dev more robust as an open source project.

## Community health

- [ ] `README.md` updated with mission and key links
- [ ] `CODE_OF_CONDUCT.md`
- [ ] `CONTRIBUTING.md`
- [ ] `SECURITY.md`
- [ ] `SUPPORT.md`
- [ ] `GOVERNANCE.md`
- [ ] `CONTRIBUTION_TIERS.md`
- [ ] `CONTENT_GUIDELINES.md`
- [ ] `DISCUSSIONS_GUIDELINES.md`
- [ ] `MODERATION_POLICY.md`
- [ ] `PUBLISHING_STRATEGY.md`

## GitHub

- [ ] branch protection on `main`
- [ ] pull requests required for every change
- [ ] at least 1 approval for small PRs
- [ ] at least 2 approvals for critical PRs
- [ ] required status checks
- [ ] GitHub Discussions enabled
- [ ] issue templates configured
- [ ] pull request template configured
- [ ] security advisories enabled
- [ ] Dependabot enabled if dependencies exist

## Publishing

- [ ] article folder documented
- [ ] article template available
- [ ] frontmatter documented
- [ ] editorial review process clear
- [ ] discussion creation documented
- [ ] editorial calendar defined
- [ ] LinkedIn templates ready

## Legal

- [ ] code license selected
- [ ] content license selected
- [ ] asset/image policy defined
- [ ] trademark/logo note defined
- [ ] clear author consent in contributing flow

## Security

- [ ] security email configured
- [ ] private vulnerability channel configured
- [ ] secrets/tokens rules defined
- [ ] dependency scanning enabled if applicable
- [ ] secret scanning enabled if available

## Moderation

- [ ] private reporting channel configured
- [ ] responsible maintainers identified
- [ ] escalation process documented
- [ ] appeal policy documented
- [ ] conflict of interest handling documented
