# dout.dev Community

dout.dev is a community/open blog.

Anyone can propose an article through fork and pull request. Article comments are managed through GitHub Discussions in the project repository.

## How it works

1. fork repository
2. add article in Markdown
3. open pull request
4. maintainers review and vote
5. if approved, article is published
6. discussion continues on GitHub Discussions

## Open mic, with care

Blog is designed as open mic: you can bring ideas, opinions, tutorials, experiences, and personal viewpoints.

There are still rules:

- content must be relevant
- Code of Conduct must be respected
- sources and copyright matter
- PRs are voted by maintainers
- comments are moderated when needed

## Becoming contributor

Requires at least 3 approved and published articles.

## Becoming maintainer

Requires at least 3 merged technical PRs on source code, excluding PRs that only involve Markdown articles.
